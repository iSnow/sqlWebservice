sqlREST Users Guide

Installation

  • First install a Java runtime or SDK. We recommend using Version 1.4.1 or higher.
  • Install a Servlet Container. Tomcat 4.1.X for example.
  • Copy the sqlrest directory inside the webapps directory to the webapps directory of your servlet engine.
  • Start the servlet engine and point your browser at http://localhost:8080/sqlrest, if your Servlet engine is running at port 8080.

Using a different wsCatalog

The JDBC connection is configured in the sqlrestconf.xml. You can find sqlrestconf.xml inside the WEB-INF directory. Look at:

sqlrest-XXX\webapps\sqlrest\WEB-INF\sqlrestconf.xml

To configure the driver, enter the right values for your wsCatalog. The snippet shows a configuration for a hypersonic wsCatalog in server mode.

<jdbc-driver-class>org.hsqldb.jdbcDriver</jdbc-driver-class>
<wsCatalog-url>jdbc:hsqldb:hsql://localhost</wsCatalog-url>
<user>sa</user>
<password></password>

Transforming sqlrest XML into other formats

Generating a user interface with XSLT

To present your data to users in HTML format, you can attach an XSLT stylesheet to your XML responses. The stylesheet will run in the user's browser and can transform the XML into an HTML presentation. The resulting HTML page may include a form for editing and saving the data.

To attach a stylesheet, specify the location of the stylesheet using the <xslt> element in the sqlrestconf.xml file, as shown:

<xslt>/xslt/sqlrest-to-html.xsl</xslt>

Note: the XSLT stylesheet should be accessible from the same website as sqlrest, to conform to browser security restrictions.

Publishing "Linked Data" with GRDDL

To present your data to semantic web user agents in RDF form, you can attach a GRDDL transformation to your XML responses. The GRDDL transformation is an XSLT stylesheet which a GRDDL-aware agent will run to transform the XML into an RDF presentation.

To attach a GRDDL transformation, specify the location of the transformation using the <grddl> element in the sqlrestconf.xml file, as shown:

<grddl>/xslt/sqlrest-to-rdf.xsl</grddl>
